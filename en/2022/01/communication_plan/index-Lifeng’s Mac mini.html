What is "Communication Plan"

  2022-01-02   LEARNING   PM Learning  Comments Word Count: 664words Read Count: 4minutes  中文版

In the “Project Management” Course, one of the topics is “Communication Plan”.

When the phrase “communication plan” is used, I feel it is pretty clear what should I do to sell the product and have better connection with the users, and among the developers.

Communication plans can help you clarify the purpose of a product launch or new initiative and officially determine the messages you want to deliver to your intended audience(s).

During my CAD daily work, the important part is to talk to the users, support them.

  • Weekly meetings
  • Feedback collecting
  • Documentation
  • etc.

Some of them are driven by the users, and some of them are driven by the product team. I never think about I can group them and call all these activities “Communication Plan”.

Prepare the “Communication Plan”

When preparing the plan, always ask ourselves these questions.

  • Why do you want to communicate with the community? (What’s your purpose?)
  • To whom do you want to communicate it? (Who’s your audience?)
  • What do you want to communicate? (What’s your message?)
  • How do you want to communicate it? (What communication channels will you use?)
  • Whom should you contact, and what should you do to use those channels? (How will you distribute your message?)

Once these questions get answered, we have a better picture for the CAD product communication.

First, during the project implementation, project manager needs to communicate with the developers to make sure the project is on track. Like check the milestone, enable the collabration, resolve the issues, etc.

Second, after involving the key users, collect the feedback, understand the usage difficulty of new CAD product.

Third, think about the documentation, not only for the end-users, but also for the development community.

Last but not the least, talk to upper management for the upcomming new product or followup revision of current product.

Convert “Communication Plan” to a document and share

Like all other plans in the project, “Communication Plan” needs to be clearly documented, and transparent to the project members and users.

Here is a table for us to easily go through the key items one by one and then put into the document.

Description Frequency Channel Audience Owner Status
Kick off meeting One time Zoom + Meeting-Room All internal stakeholders and project team members Project Manager@PM Planned
Status updates Weekly Zoom + Meeting-Room All project team members Project Manager@PM Planned
Milestone and deliverable updates Monthly Zoom All project team members, stakeholders optional Project Manager@PM Planned
Project stand-up (check-in) Daily Slack Project team Project Team Planned
Weekly User Synch-up Weekly Zoom + Meeting-Room Customers and project team Customers and Project Team Planned

Use python to generate ics files

The goal for the python script is to generate different kinds of meeting place holders for a project.

  • Kick off meeting
  • Weekly meeting to track the project status
  • Milestone Review meetings
  • Meetings to resolve the conflicts and issues
  • Project closing meeting

To test the following function

# Install iCalendar
pip install icalendar

# Run the function
python create_ics.py

# Test example.ics
# Double click example.ics


#!env python3

from icalendar import Calendar, Event, vCalAddress, vText
import pytz
from datetime import datetime
import os
from pathlib import Path

# Creating icalendar/event
event = Event()
event.add('summary', 'Python meeting about calendaring')
event.add('dtstart', datetime(2021, 1, 12, 8, 0, 0, tzinfo=pytz.utc))
event.add('dtend', datetime(2021, 1, 12, 10, 0, 0, tzinfo=pytz.utc))
event.add('dtstamp', datetime(2021, 1, 12, 0, 10, 0, tzinfo=pytz.utc))

# Adding Organizer
organizer = vCalAddress('MAILTO:organizer@example.com')
organizer.params['cn'] = vText('Organizer')
organizer.params['role'] = vText('CHAIR')
event['organizer'] = organizer

# Adding attendee
attendee = vCalAddress('MAILTO:foobar@example.com')
attendee.params['cn'] = vText('Foo Bar')
attendee.params['ROLE'] = vText('REQ-PARTICIPANT')
event.add('attendee', attendee, encode=0)

# Adding location
event['location'] = vText('Zoom Link')

# Adding events to calendar
cal = Calendar()
cal.add_component(event)

directory = str(Path(__file__).parent.parent) + "/"
# print(directory)
f = open(os.path.join(directory, 'example.ics'), 'wb')
f.write(cal.to_ical())
f.close()
