Use tree command to list the structure

Jan 3, 2021 4 min read csh RUI tree

tree command is helping to show the structure of folders and files.

Real Case

When discussing the folder structure via video meeting, express the structure is a little bit challenge if we have a lot of levels and files. How to make the folder and file structure clear when we present something?

. means in current dir. and we can use pwd to replace it.

➜  tree -a --dirsfirst
.
├── test -> test_3
├── test_1
│   └── file_1
├── test_2
│   └── file_2
├── test_3
│   └── file_3
└── .conf
4 directories, 4 files

➜  tree `pwd` -a --dirsfirst
/Users/steve/demo
├── test -> test_3
├── test_1
│   └── file_1
├── test_2
│   └── file_2
├── test_3
│   └── file_3
└── .conf

4 directories, 4 files

I will show some flags’ usage which I am using a lot. The options are very similar to the ls command.

-a, list all the files including the hidden files. In the CAD development, some files are for debugging purpose or for one-time configuration, and they are hidden. To list the hidden files, -a must be present.

-d, only list the folder, In some cases, the file list is huge but we only care about the folder structure. The folder will tell the clear structure

-L, controls the levels, sometimes, I don’t want to dive into all the folder levels and I only want to show 2 levels, then use -L 2.

-t, sort by last modified time. The flag is useful because when we want to see some new files generated recently, we can easily find them at the end of the tree view.

Another useful function is HTML or JSON output. In these days, HTML document is very popular and we can use tree to output the html code or json file to use them in our web applications.

tree `pwd`
tree `pwd` -a
tree `pwd` -a --dirsfirst
touch demo_2/file_2_2
tree `pwd` -a -t --dirsfirst
tree `pwd` -a -t -L 1 --dirsfirst
tree `pwd` -a -t -d -L 1 --dirsfirst
