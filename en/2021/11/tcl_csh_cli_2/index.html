Project Goals (tcl-csh-cli)

Nov 28, 2021 2 min read CLI Csh PM Tcl

As a project manager, one very important task is to learn, understand, broadcast the project goal correctly and efficiently. This must be documented and put a place, so that project manager can go through it in the meeting and the team can refer to the document any time.

The solution could be dropbox, sharepoint, onenote, git, ftp, etc.

Goal of “tcl-csh-cli”

Let’s look at the goal of “tcl-csh-cli” project.

  • Major Goal
    • Create a framework using tcl and csh to handle the complex design requirement.
  • Milestones
    • Create csh cli interface with Tcl (Phase 1) in Dec. 2021
      • To enable the development of tcl
      • To enable the test friendly concept
      • The framework concept, like hotfix, portability, etc.
    • Implement tcl function and connect with csh (Phase 1) in Jan. 2022
      • The tcl function must be for VLSI design consume.
      • The framework concept, like plugin, hotfix, portability, etc.
      • To enable the test friendly concept
    • Create csh cli interface with Tcl (Phase 2) in Feb. 2022
      • More details, like interuption handling, extension (plugin), etc.
    • Implement tcl function and connect with csh (Phase 2) in Mar. 2022
      • More details, like interuption handling, extension (plugin), etc.
    • Document and Training during the whole project.
      • Writing the training material
  • Some details
    • Source code version control, using github.
    • Put all the document in the docs folder of git repository.

Recap

It is important to share the main goal with all the team members and make sure we have the same target.

And also it is important to define the milestones, we may need to adjust in the future, but make sure we have a plan when we start the project.
