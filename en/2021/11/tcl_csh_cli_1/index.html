Project Kick-off (tcl-csh-cli)

  2021-11-24   Project   CLI Tcl Csh PM  Comments Word Count: 437 words Read Time: 3 minutes

This is a project used to demostrate the project management process and also it is my first project which could benefit the IC design community.

I will build a course to finish this project through several months. And the project name is “Build Cli with tcl and csh”.

The project will be split to several sections in a course to tell some key technology stuff and program management skills.

Project Initialzation

First question, how does this project come out?

Usually, for an application, we have Graphic User Interface and Command User Interface.

In EDA industry, tcl is still a very important glue language. I wrote a post to share some thought why tcl is a fit choice for EDA industry.

Nowdays, python is more and more popular, and to use python as the main interface language is possible, but not just one or two years task.

Tcl is still the major language for VLSI industry.

Intention

Tcl is very flexible, and too flexible to error out. We need a framework to constrain certain use model. This is the initiative of “tcl-csh-cli” project.

Prepare the kick-off meeting

  • Check if the meeting is intended to involve all the developers or just some key stakeholders and developers. Make sure you can get more good support in the meeting. (pre-kickoff talk with people)
  • Make a decision, what collabration tool will be used, like Jira, Asana, Focalboard, etc.
  • Draft the schedule and milestone

Kick-Off meeting minutes

  • Date: 11/24/2021
  • Attendees: Steve, Jane, Peter
  • Topic: “tcl-csh-cli” kick-off meeting
  1. Will this project be alive for long term? Maybe not, this is more like a demostrate project, we can keep maintaining it if we find a good path. And usually, there are a lot of commercial solutions.
  2. Resource planning: I will create several virtual roles to tell some development stories and also show case how different teams or different poeple work with each other.
  3. Do we need GUI? Yes, we need, but must be built on top of CLI and the priority is low.
  4. What is the cost and profit? Time is the main costs for this project, and profit is to make friends, and earn money for support and plugin sale.
  5. Define deliveries: A framework for developers to develop the flows.
  6. Schedule and Milestone: The entire project is splited to three phases, one is the basic feature, and other two are advanced feature.
  7. Collect user feedbacks: Reserve some meetings to collect user’s feedback.
